# PACT — Protocol for Agentic Commerce and Trust

> *"MCP solved how agents connect to tools. PACT solves how agents transact in markets."*

## What is PACT?

AI agents are becoming digital workers. They draft code, run campaigns, manage
infrastructure — and increasingly, they need to spend money to do their jobs.

The problem: there is no standard for proving that an agent's purchase decision
was legitimate. Existing agentic payment solutions (Stripe, Mastercard, Visa,
Google) solve the human→agent→merchant flow. Nobody has built the
**agent→agent layer** — where an agent autonomously procures compute, software,
contractors, physical hardware, or even human employees, with cryptographic
proof that every purchase was authorised.

PACT is that standard.

## What PACT does

PACT sits *above* payment rails (Stripe, stablecoins, bank APIs). It does not
move money. It answers one question before money moves:

> **Was this agent authorised to make this specific purchase?**

It answers it with a cryptographic **Proof of Valid Purchase (PVP)** — a signed
object generated by the purchasing agent that proves:

1. The agent is who it claims to be (identity)
2. The agent is authorised to make this category of purchase (mandate)
3. The purchase satisfies the agent's constraints (logical validity)
4. The vendor is verified or risk is acknowledged (trust tier)

If the proof passes, the payment rail proceeds. If it fails, the transaction is
rejected and the agent escalates to its human principal.

## Three components

| Component | What it does |
|---|---|
| **PVP (Proof of Valid Purchase)** | Cryptographic certificate of a legitimate agent purchase decision |
| **SDK** | Drop-in Python library — give any agent commercial capability in 30 minutes |
| **Registry** | Agent identity, vendor trust tiers, and on-chain reputation |

## Run the reference implementation

Requires Python 3.8+ and one dependency:

```bash
pip install cryptography
python pact_reference_v01.py
```

This runs a complete A2A transaction end-to-end using real Ed25519 cryptography.
No internet. No API keys. No money moves. You will see a full PVP generated,
signed, and verified in your terminal.

published early to attract collaboration from developers who are building agent

If you're building something that involves agents spending money and you're
